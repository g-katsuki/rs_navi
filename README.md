# rs_navi
ロボットが自律走行を行うためのライブラリ(提出用)

## 概要
　ロボットの自律走行を行うには障害物を回避しつつ与えられた目的地点へ向かう進行方向の計算を, マップを更新しながら一定間隔ごとに繰り返す必要があります.
　本来このライブラリは目的地点の決定, 障害物検知, 最適な進行方向の計算の３つで構成されていますが, モジュール単位で分担しチーム開発を行っているため, 自分が作成したobstacle_detector* , potential* の２つとなっています.

## obstacle_detector.cpp
　LiDARから取得した点群情報から障害物を検出し障害物の座標情報をrs_commonで定義されたObstacleに格納することを目的としています.
　点群一つ一つを別の障害物としていては後の障害物回避での処理が多くなってしまい自律走行をするうえで他の処理に影響を及ぼしてしまいます. その為, 点群をまとめて直線と矩形で表現することにより情報量を減らしつつ扱いやすい情報を格納するようにしています. 全て矩形で表現してしまうと大きい障害物を矩形にしたとき点群を含まない多くの領域が矩形内に含まれてしまう可能性があるので直線の表現も採用しています.

## potential.cpp
　矩形と直線で表された障害物情報とロボットの現在位置情報から, 各障害物上の最も近い一点に正のポテンシャル値を与え, 目標地点には負のポテンシャル値を与えるようにしてこれらから構成されるポテンシャル場によって最適な進行方向を決定します.
